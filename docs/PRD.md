# ArchFocus - 小红书AI运营系统 产品需求文档

## 1. 产品概述

**产品名称**：ArchFocus（小红书 AI 运营助手）

**产品定位**：面向小红书个人博主的单人多账号 AI 运营 SaaS 平台，帮助博主完成从选题、创作到数据复盘的全流程运营。

**目标用户**：小红书个人博主，拥有 1-3 个垂直账号，需要提升内容创作效率和数据分析能力。

**核心价值**：通过 AI 辅助选题、文案生成、配图制作，以及基于真实数据的笔记分析，降低内容创作门槛，提升账号运营效果。

- **右上角账户选择器**：创作、笔记库、数据分析页面均有账户下拉框，默认记忆上次选择

### 2.3 创作（Content Creation）

四步向导流程：选题 → 文案 → 配图 → 发布。

#### 2.3.1 选题（Step 1）
- **发布账号**：以文本形式显示当前选中账户（非下拉框）
- **数据源**：多选 - 联网搜索、小红书行业热点词
- **主题输入**：AI 推荐主题（调用后展示多个推荐卡片，点击选中）或手动输入笔记主题
- **推荐主题展示**：标题 + 推荐理由（基于热点和人设）

#### 2.3.2 文案生成（Step 2）
- **文案模型选择**：Kimi / MiniMax（默认）/ DeepSeek
- **写作要求输入**：textarea 输入具体要求
- **提示文字**："可直接点击文案内容进行修改"
- **文案展示**：contenteditable 区域，可内联编辑
- **审核流程**：
  - "通过，去配图" → 进入 Step 3
  - "重新生成" → 打开反馈弹窗，输入修改意见后重新生成

#### 2.3.3 配图生成（Step 3）
- **配图类型**：两种模式切换
  - **AI+提示词**：AI 生图提示词（可修改）+ 绘图模型选择（Image 2.0 / Seedream）+ 生图数量（默认 1 张）
  - **HTML→截图**：笔记内容预览 + 5 种风格选择（杂志风/奶油暖调/森林绿/瑞士极简/多巴胺）+ 自动按内容长度计算页数（≤100 字=1 张，≤300=2，≤600=3，≤1000=4，>1000=5）

#### 2.3.4 发布确认（Step 4）
- **笔记标题**：只读展示 + 复制按钮
- **笔记内容**：只读展示 + 复制按钮
- **笔记配图**：每张图一行（封面/图2-图6），附带每张图片的复制按钮
- **发布操作**：
  - 立即发布
  - 定时发布（弹出定时选择弹窗）
- **定时发布弹窗**：
  - 快捷选择：明天晚上7点 / 后天晚上7点
  - 自定义日期 + 时间
  - 提前10分钟邮件提醒开关 + 邮箱输入

## 5. AI 架构：工作流 → Agent → Skills → Tools

### 5.1 架构总览

```
                    ┌─────────────────────────────────┐
                    │        用户交互层 (React UI)       │
                    └──────────────┬──────────────────┘
                                   │
                    ┌──────────────▼──────────────────┐
                    │        LangGraph 工作流引擎        │
                    │  固定 DAG，定义节点顺序与数据流转    │
                    │                                  │
                    │  ┌──────────┐  ┌──────────┐     │
                    │  │  节点 A   │→│  节点 B   │     │
                    │  │ (函数/API)│  │ (Agent)   │     │
                    │  └──────────┘  └─────┬────┘     │
                    │                     │          │
                    │  ┌──────────┐  ┌────▼─────┐    │
                    │  │  节点 C   │←│  节点 D   │    │
                    │  │ (Agent)   │  │ (函数/API)│    │
                    │  └──────────┘  └──────────┘     │
                    └─────────────────────────────────┘
                              │
                    ┌─────────▼───────────────────────┐
                    │    Agent 节点运行时               │
                    │  「小红书文案大师」在工作流节点内    │
                    │  执行 Skill / 调用 Tool / 做决策   │
                    └──┬───────┬───────┬──────┬───────┘
                       │       │       │      │
              ┌────────▼──┐ ┌──▼───┐ ┌─▼───┐ ┌▼───────┐
              │ 文案生成   │ │去AI位│ │配图  │ │质量判断 │
              │ (Skill)   │ │(Skill)│ │(Skill)│ │(Skill) │
              └─────┬─────┘ └──┬───┘ └──┬───┘ └──┬─────┘
                    │          │        │        │
              ┌─────▼──────────▼────────▼────────▼─────┐
              │        工具层 (Tools)                    │
              │  LLM  │  mmx search  │  mmx image  │ 零克查词 │
              └────────────────────────────────────────┘
```

**层级说明**：

| 层级 | 角色 | 说明 |
|------|------|------|
| **工作流** | 编排者 | LangGraph 定义的固定 DAG，决定节点执行顺序和数据流转 |
| **Agent** | 执行者 | 作为工作流中的特定节点类型，负责调用 Skill、使用 Tool、做判断 |
| **Skill** | 能力单元 | 封装好的原子能力函数，Agent 通过 Tool Calling 调用 |
| **Tool** | 外部执行器 | 非 AI 的外部工具（搜索引擎、API、浏览器自动化） |

工作流本身是固定的有向无环图，不依赖 Agent 做路径决策。Agent 只在特定节点内发挥作用：调用 Skill 完成生成任务，或做质量判断。

### 5.2 Agent：小红书文案大师

**定位**：作为工作流节点内的智能执行单元，负责执行 Skill 调用和做质量判断。

**两种使用方式**：

**方式一：Skill 执行节点**
- 工作流到达文案生成节点时，实例化 Agent
- Agent 接收节点输入（选题、人设、要求等），调用「文案生成」Skill
- Skill 内部调用 LLM + 联网搜索等 Tool 完成生成
- Agent 返回生成结果，节点结束

**方式二：决策判断节点**
- 工作流到达质量审核节点时，实例化 Agent
- Agent 接收待审核内容，从多个维度判断（是否达标、敏感词是否严重、是否需要修改）
- Agent 输出决策结果，工作流根据结果走不同分支

**决策边界**：

| 决策场景 | Agent 自主 | 需要用户确认 |
|----------|-----------|-------------|
| 联网搜索结果筛选与摘要 | ✅ | |
| 去AI位强度调整 | ✅ | |
| 配图风格推荐 | ✅（基于人设推断） | |
| 敏感词自动替换 | | ✅ 标记出来由用户决定 |
| 最终文案是否通过 | | ✅ |
| 配图是否满意 | | ✅ |
| 文案修改后是否重新生成 | | ✅ 用户反馈后触发 |
|----------|-----------|-------------|
| 是否需要联网搜索当前选题 | ✅ | |
| 选择哪个 LLM 模型 | ✅（从用户偏好中选） | |
| 去AI位的强度 | ✅ | |
| 配图风格选择 | ✅（基于人设推断） | |
| 敏感词自动替换 vs 标记 | | ✅ 标记出来由用户决定 |
| 最终文案是否通过 | | ✅ |
| 配图是否满意 | | ✅ |

### 5.3 Skills（AI 能力单元）

Skill 是 Agent 可调用的原子能力。每个 Skill 有明确的输入/输出定义，Agent 通过 Tool Calling 调用。

系统由以下离散的 AI Skill 组成：

| Skill | 说明 | 状态 |
|-------|------|------|
| **小红书选题生成** | 基于账号人设、联网搜索、热点词，生成 5 个推荐笔记主题 | ✅ 已定义 |
| **文案生成** | 基于选题、人设、写作要求，调用 LLM 生成小红书文案 | ✅ 已定义 |
| **去AI位** | 对 AI 生成文案进行"去 AI 化"润色，消除模板化表达、添加口语细节，使其更像真人原创 | ✅ 已定义 |
| **AI 配图生成提示词** | 根据笔记文案自动生成 AI 绘图提示词，支持 6 种内容类型和 5 种视觉风格 | ✅ 已定义 |
| **质量判断** | 对文案进行质量审核、敏感词标记等决策判断 | ✅ 已定义 |

#### 5.3.1 小红书选题生成
- **输入**：账号人设（定位 + 目标受众 + 人设描述）+ 联网搜索（选填）+ 小红书热点词（选填）
- **模型**：MiniMax（默认）/ Kimi / DeepSeek（用户可选）
- **输出**：5 个推荐主题，每个主题包含标题 + 推荐理由
- **交互**：Human-in-the-Loop — 用户从 5 个主题中选择一个，或手动输入自己的主题

#### 5.3.2 文案生成
- **说明**：基于选题、人设、写作要求，调用 LLM 生成小红书文案。Skill 参考 `copywriting/SKILL.md`（通用的文案撰写框架：AIDA/PAS/FAB 结构、标题公式、CTA 设计、情感触发、异议处理）
- **Skill 文件**：`copywriting/SKILL.md`
- **输入**：选题主题、账号人设信息（定位/受众/描述）、写作要求（语气/字数/重点）、数据源上下文（联网搜索结果/热点词）
- **模型**：MiniMax（默认）/ Kimi / DeepSeek（用户可选）
- **写作框架**：Agent 根据内容类型选择合适的框架
  - **AIDA** — 吸睛标题 → 引发兴趣 → 展示效果 → 引导行动（适用于种草、好物推荐）
  - **PAS** — 点出问题 → 加剧痛点 → 给出方案（适用于痛点驱动型内容）
  - **FAB** — 功能 → 优势 → 利益（适用于产品对比、测评）
- **输出**：完整小红书笔记文案（标题 + 正文 + 标签）
- **交互**：Human-in-the-Loop — 用户可编辑修改，不满意可反馈重新生成

#### 5.3.3 去AI位
- **说明**：对 AI 生成文案进行"去 AI 化"润色，消除 LLM 输出痕迹。Skill 参考 `ai-humanizer/SKILL.md`（24 种模式检测器 + 500+ AI 词汇库 + 统计信号分析）
- **Skill 文件**：`ai-humanizer/SKILL.md`
- **输入**：AI 生成的原始文案
- **处理策略**：

  **24 种模式检测（按类别）：**

  | 类别 | 模式 | 示例 |
  |------|------|------|
  | 内容 | 重要性膨胀 | "标志着...的关键时刻" |
  | 内容 | 提及媒体充数 | 罗列媒体名称但无具体观点 |
  | 内容 | 表层 -ing 分析 | "展示着...反映着...突显着..." |
  | 内容 | 推销式语言 | "坐落在"、"令人惊叹"、"著名" |
  | 内容 | 模糊引用 | "专家认为"、"研究表明" |
  | 内容 | 套路化挑战 | "尽管面临挑战...仍在蓬勃发展" |
  | 语言 | AI 高频词（3 级） | 见下方词汇库 |
  | 语言 | 回避系动词 | 用"充当"、"拥有"替代"是"、"有" |
  | 语言 | 否定排比 | "不仅是 X，更是 Y" |
  | 语言 | 三点法则 | "创新、灵感、洞察" |
  | 语言 | 同义替换循环 | "主角...主人公...核心人物..." |
  | 语言 | 假性范围 | "从大爆炸到暗物质" |
  | 风格 | 破折号滥用 | 过多—破折号—打断句子 |
  | 风格 | 加粗滥用 | **机械式**加粗**到处都是** |
  | 风格 | 行内标题列表 | "- **主题：** 这里是讨论内容" |
  | 风格 | 大写标题 | 每个单词都大写 |
  | 风格 | emoji 滥用 | 🚀💡✅ 点缀专业文本 |
  | 风格 | 弯引号 | 使用""而不是"" |
  | 通信 | 聊天机器人痕迹 | "希望这能帮到你！"、"如有问题..." |
  | 通信 | 截止期免责声明 | "根据我最后一次训练..." |
  | 通信 | 讨好语气 | "好问题！"、"说得太对了！" |
  | 填充 | 废话短语 | "为了"、"由于这样一个事实"、"在这一点上" |
  | 填充 | 过度模糊限定 | "可能也许大概" |
  | 填充 | 套路化结尾 | "未来充满希望"、"激动人心的时刻即将到来" |

  **AI 词汇库（三级）：**

  - **一级（致命暴露）**：delve、tapestry、vibrant、crucial、comprehensive、meticulous、embark、robust、seamless、groundbreaking、leverage、synergy、transformative、paramount、multifaceted、myriad、cornerstone、reimagine、empower、catalyst、invaluable、bustling、nestled、realm
  - **二级（高频嫌疑）**：furthermore、moreover、paradigm、holistic、utilize、facilitate、nuanced、illuminate、encompasses、catalyze、proactive、ubiquitous、quintessential
  - **短语**："在当今的数字时代"、"值得一提的是"、"起着关键作用"、"是...的见证"、"在...领域中"、"深入探讨"、"利用...的力量"、"踏上一段旅程"

  **统计信号检测：**

  | 信号 | 人类水平 | AI 水平 | 说明 |
  |------|---------|---------|------|
  | Burstiness（突发性） | 高 (0.5-1.0) | 低 (0.1-0.3) | 人类写作有起伏，AI 节奏均匀 |
  | Type-token ratio（词汇多样性） | 0.5-0.7 | 0.3-0.5 | AI 重复使用相同词汇 |
  | 句长变异系数 | 高 | 低 | AI 句子长度均匀 |
  | 三元组重复率 | 低 (<0.05) | 高 (>0.10) | AI 重复 3 词短语 |

  **核心改写原则：**

  1. **像真人一样写，不要像新闻稿** — 自由使用"是"、"有"，避免"充当"等虚张声势的表达
  2. **加入个性** — 有观点，对事实有反应而不只是复述；长短句交错
  3. **精简冗余** — "为了" → "为了"；"值得一提的是" → 直接说
  4. **去除填充** — 删除聊天机器人痕迹、套路化结尾、讨好语气

- **输出**：更自然的真人风格文案
- **交互**：用户可对比去AI位前后的差异，手动进一步调整

#### 5.3.4 AI 配图生成提示词
- **说明**：根据笔记文案自动生成适配的 AI 绘图提示词
- **Skill 文件**：`.claude/skills/aigc-image-prompt/SKILL.md`
- **输入**：笔记标题 + 正文内容 + 标签（可选）+ 账号人设风格（可选）
- **处理逻辑**：
  1. 分析文案内容，识别配图类型（cover/tutorial/data/compare/list/lifestyle）
  2. 根据类型应用对应的构图规则
  3. 根据账号人设推断或用户指定选择视觉风格
  4. 组合通用约束 + 风格特定约束
  5. 输出结构化提示词及生成参数
- **输出**：完整 AI 绘图提示词 + 生成参数（size/resolution/style）
- **覆盖内容类型**：封面型、教程型、数据型、对比型、清单型、生活型
- **支持视觉风格**：ins 风、极简风、治愈温暖风、复古风、清新自然风

### 5.4 核心工作流（Workflows）

工作流是 LangGraph 定义的**固定 DAG**，节点执行顺序预先确定。节点分为两种类型：

| 节点类型 | 说明 | 示例 |
|----------|------|------|
| **函数节点** | 直接执行外部 API 调用，无 AI 推理 | `mmx search` 调用、零克查词 API 调用 |
| **Agent 节点** | 实例化「小红书文案大师」Agent，调用 Skill 或做决策 | 文案生成、质量判断 |

工作流图例：`[函数]` = 函数节点，`[Agent]` = Agent 节点，`{用户}` = Human-in-the-Loop 节点

#### 5.4.1 文案生成工作流

```
[Agent 小红书选题生成]
  │ Agent：「小红书文案大师」
  │ Skill：小红书选题生成
  │ Tool：LLM + mmx search（可选）
  │ 输入：账号人设（定位+受众+描述）+ 联网搜索（选填）+ 小红书热点词（选填）
  │ 输出：5 个推荐主题（标题 + 推荐理由）
  │
  ▼
[选题去重]  ── 函数节点
  │ Tool：笔记库查询（5.5.3）
  │ 执行：getAllTitles(accountId) → 获取已发布 + 待发布笔记标题列表
  │ 逻辑：将 5 个推荐主题与笔记库标题做语义相似度对比（余弦相似度 > 0.85 标记重复）
  │
  ├── 全部重复 ──→ 回到 [Agent 小红书选题生成]
  │                 输入追加"请避免以下已有主题：{笔记库标题列表}"
  │                 最多重试 2 次，仍全部重复则终止并提示用户手动选题
  │
  ├── 部分重复 ──→ 保留不重复主题，仅对重复部分请求补充生成 → 合并为 5 个主题
  │
  └── 无重复 ────→

{用户选择主题}
  │ 从 5 个推荐主题中点击选择一个，或手动输入主题
  │
  ▼
[联网搜索]  ── 函数节点（可选，基于所选主题搜索实时内容）
  │ 执行：mmx search "<主题> 小红书" --output json --quiet --non-interactive
  │ 输出：搜索结果 JSON（标题、摘要、来源链接）
  │
  ▼
[Agent 文案生成]
  │ Agent：「小红书文案大师」
  │ Skill：文案生成
  │ Tool：LLM（MiniMax 默认 / Kimi / DeepSeek）
  │ 输入：选中的主题 + 账号人设 + 写作要求 + 联网搜索结果（可选）
  │ 输出：文案初稿（标题 + 正文 + 标签）
  │
  ▼
[Agent 去AI位]
  │ Agent：「小红书文案大师」
  │ Skill：去AI位
  │ Tool：LLM
  │ 输入：文案初稿
  │ 输出：真人风格文案
  │
  ▼
[敏感词检测]  ── 函数节点
  │ 执行：POST https://ci.lingke.pro/api/check
  │ 输出：{ result: boolean, illegal_words: [{ word, type }] }
  │
  ├── 通过 ──────→ {用户审核}
  │                   ├── 通过 → 进入配图步骤
  │                   └── 不通过 → 用户输入修改意见
  │                                   │
  │                                   ▼
  │                                [Agent 文案生成]（二次生成）
  │                                  ├── 输入：原生成文案 + 用户修改意见 + 账号人设
  │                                  └── Agent 在原文案基础上按反馈修改，非重新生成
  │                                   │
  │                                   ▼
  │                                [Agent 去AI位] → [敏感词检测] → {用户审核}
  │
  └── 命中敏感词 ──→ [Agent 标记] → {用户修改} → 回到 [敏感词检测]
```

**Human-in-the-Loop 节点**：选择主题、文案编辑修改、审核通过/拒绝（带原文案反馈二次生成）、敏感词处理、选题去重（全部重复时终止提示）

#### 5.4.2 配图生成工作流

```
{用户进入配图步骤}
  │
  ▼
{用户选择配图类型}
  │
  ├── AI提示词生图
  │     │
  │     ▼
  │   [Agent 配图提示词生成]

  │   │ 输入：笔记文案（标题 + 正文 + 标签）+ 账号人设风格
  │   │ 处理：Agent 读取 aigc-image-prompt Skill，分析文案内容类型
  │   │       → 识别 cover/tutorial/data/compare/list/lifestyle
  │   │       → 应用对应构图规则 + 风格预设
  │   │       → 组合完整提示词
  │   │ 输出：结构化 AI 绘图提示词 + 生成参数
  │     │
  │     ▼
  │   {用户选择模型} ── 用户选择后进入生图
  │   │ Image 2.0（默认）/ Seedream
  │   │ 可用时展示预览（提示词文本）

  │   │ 执行：通过 APIMart API 调用对应模型（POST /v1/images/generations）
  │   │       gpt-image-2（默认）/ doubao-seedream-5-0-lite
  │   │      （minimax-image-01 直接调用 MiniMax API）
  │   │ 输入：绘图提示词 + 模型参数（size/resolution/style）+ 生图数量

  │     │

  │     ├── 满意 → 进入发布步骤
  │     └── 不满意 → 返回 [Agent 配图提示词生成]（用户提供修改方向）
  │
  └── HTML→截图
        │
        ▼
      [Agent HTML生成]
      │ Agent：「小红书文案大师」
      │ 输入：笔记文案 + 配色方案（从人设推断或用户选择）
      │       风格：杂志风 / 奶油暖调 / 森林绿 / 瑞士极简 / 多巴胺

      │       → 标题区、正文区、装饰元素按风格模板布局

      │   ≤1000 字 = 4 张

        ▼

        └── 不满意 → 返回 [Agent HTML生成]（用户反馈修改方向/切换风格）
```

- 选择配图类型（AI 提示词生图 / HTML→截图）
- 选择绘图模型（Image 2.0 / Seedream）
- 确认/拒绝配图结果，不满意可反馈修改方向后重新生成

### 5.5 工具集成

#### 5.5.1 联网搜索 — mmx-cli

**工具**：MiniMax MMX-CLI `mmx search`

**用途**：在选题阶段和文案生成阶段搜索实时信息，为主题推荐和文案撰写提供上下文参考。

**集成方式**：
```bash
# 搜索热门话题
mmx search "小红书 夏季美妆 热门话题" --output json --quiet --non-interactive

# 搜索特定主题的实时讨论
mmx search "油皮持妆 2026 热门笔记" --output json --quiet
```

**输出格式**：JSON 结构化数据，包含标题、摘要、来源链接

#### 5.5.2 敏感词检测 — 零克查词

**工具**：零克查词（https://ci.lingke.pro）

**用途**：检测小红书文案中的敏感词、违禁词、广告词，规避平台限流。

**集成方案（两种）**：

**方案一：Cookie API 方式（推荐优先使用）**
- **端点**：`POST https://ci.lingke.pro/api/check`
- **认证**：携带从浏览器抓取的会话 Cookie
- **请求格式**：`application/x-www-form-urlencoded`，参数 `text=文案内容`
- **字符上限**：约 5000 字符
- **响应格式**：
  ```json
  {
    "result": false,
    "illegal_words": [
      { "word": "违禁词", "type": "违禁" }
    ]
  }
  ```
- **集成策略**：
  - 定时刷新 Cookie（可配合浏览器自动化保持会话有效）
  - 检测失败时重试（Cookie 过期 → 重新获取 → 重试请求）

**方案二：浏览器自动化兜底**
- 若 Cookie API 方式不可靠，采用 Puppeteer/Playwright 模拟浏览器操作
- **流程**：
  1. 启动无头浏览器，打开 `https://ci.lingke.pro`
  2. 输入文本，点击"一键检测"
  3. 读取检测结果（DOM 解析）
- **适用场景**：Cookie 频繁过期、API 端变更导致方式一不可用时的降级方案

**检测流程**：
```
文案 → 零克查词检测 → result=true（通过）→ 继续流程
                       → result=false（命中）→ 返回违规词列表 → 前端高亮标记 → 用户修改 → 重新检测
```

#### 5.5.3 笔记库查询

**工具**：专用数据库查询接口

**用途**：供工作流节点查询当前账户的笔记库数据。仅开放按需查询，不暴露原始数据库写入权限。

**接口定义**：

| 接口 | 参数 | 返回 | 调用方 |
|------|------|------|--------|
| `getPublishedTitles(accountId)` | accountId | `string[]` — 已发布笔记标题列表 | [选题去重] 节点 |
| `getPendingTitles(accountId)` | accountId | `string[]` — 待发布笔记标题列表 | [选题去重] 节点 |
| `getAllTitles(accountId)` | accountId | `[已发布标题列表, 待发布标题列表]` | [选题去重] 节点（合并去重） |

**数据模型**：查询结果映射自 `Note` 表的 `title` 字段（参见第 4.2 节），不返回全文 content 或其他字段。

**集成约束**：
- 只读查询，不提供写入接口
- 按 `accountId` 过滤，无权跨账户访问
- 标题列表上限 500 条，超出截断
- 语义去重逻辑在工作流节点内执行，工具层只负责返回标题列表

**去重逻辑概览**（在 [选题去重] 节点内执行）：
```
标题列表 → 逐一计算语义向量 → 与生成主题做余弦相似度
相似度 > 0.85 → 标记为重复
```

## 6. AI 编排架构（LangGraph + Agent）

```
                    ┌─────────────────────────────────┐
                    │        用户交互层 (React UI)       │
                    └──────────────┬──────────────────┘
                                   │
                    ┌──────────────▼──────────────────┐
                    │         LangGraph Runtime         │
                    │                                   │
                    │  ┌─── 文案生成工作流 ──────────┐  │
                    │  │ [联网搜索] → [Agent:文案] →  │  │
                    │  │ [Agent:去AI位] → [敏感词检测] │  │
                    │  └────────────────────────────┘  │
                    │                                   │
                    │  ┌─── 配图生成工作流 ──────────┐  │
                    │  │ [Agent:提示词] → [mmx image] │  │
                    │  │          或                   │  │
                    │  │ [渲染+截图]                   │  │
                    │  └────────────────────────────┘  │
                    │                                   │
                    │  Human-in-the-Loop：关键节点暂停    │
                    └─────────────────────────────────┘
                              │
                    ┌─────────▼───────────────────────┐
                    │  Agent 节点：「小红书文案大师」    │
                    │  - 文案生成时：调用 Skill + Tool  │
                    │  - 质检时：判断质量、标记问题      │
                    └──┬───────┬───────┬──────┬───────┘
                       │       │       │      │
              ┌────────▼──┐ ┌──▼───┐ ┌─▼───┐ ┌▼───────┐
              │ 文案生成   │ │去AI位│ │配图  │ │质量判断 │
              │ (Skill)   │ │(Skill)│ │(Skill)│ │(Skill) │
              └─────┬─────┘ └──┬───┘ └──┬───┘ └──┬─────┘
                    │          │        │        │
              ┌─────▼──────────▼────────▼────────▼─────┐
              │        工具层 (Tools)                    │
              │  LLM  │  mmx search  │  mmx image  │ 零克查词 │
              └────────────────────────────────────────┘
```

**核心设计原则**：

| 原则 | 说明 |
|------|------|
| **工作流是固定 DAG** | 节点顺序和数据流转由 LangGraph 图定义，Agent 不决定"下一步做什么" |
| **Agent 是节点内智能** | Agent 只在自己的工作流节点内发挥作用，不跨节点编排 |
| **Skill 是 Agent 的工具** | Agent 通过 Tool Calling 调用 Skill 完成具体任务 |
| **函数节点直连 Tool** | 不需要 AI 推理的操作（如 API 调用）走函数节点，不经过 Agent |
| **Human-in-the-Loop** | 关键节点暂停等待用户输入，Agent 根据用户反馈调整输出 |
